# MKPSE 运行日志智能诊断说明书 (Log Diagnostic Manual)

> **文档用途**：本手册用于解析 MKPSE (MKP Support Electron) 产生的极简状态码日志。无论是开发者人工排查，还是将日志交由 AI 进行自动化错误诊断，均以此手册为唯一解释标准。

## 1. 日志格式规范 (Log Protocol)
日志的标准输出格式为：`[时间戳] [类型+状态码] 附加参数`
* **示例**：`[14:20:05] [O202] p:a1mini, v:standard`
* **类型前缀**：
  * **`O` (Operation)**：代表用户的常规交互或系统的正常运行节点（行为面包屑）。
  * **`E` (Error)**：代表发生错误、异常或阻断性问题。
* **附加参数**：采用极简的 `key:value` 键值对拼接，逗号分隔。

## 2. 模块定义 (Modules)
状态码按百位段划分为 6 大核心模块：
* **1xx (SYS)**：系统与生命周期 (初始化、更新)
* **2xx (UI)**：界面与交互 (点击、切换、引导)
* **3xx (IO)**：本地文件读写 (预设JSON、日志、模型复制)
* **4xx (NET)**：网络与云端 (清单拉取、文件下载)
* **5xx (CORE)**：核心数据与引擎 (Z轴计算、AST解析、Gcode处理)
* **6xx (EXT)**：外部系统调用 (切片软件、文件管理器、系统通知)

## 3. 完整状态码字典与排障指南 (Dictionary & Troubleshooting)

### 模块 1：系统与生命周期 (SYS - 1xx)
| 代码 | 定义 (Action) | 说明与可能原因 (Causes) | 排障建议 (Fix) |
| :--- | :--- | :--- | :--- |
| **O101** | App init start | 软件启动，主进程开始初始化。 | 无需处理，正常流程。 |
| **O102** | App init done | 软件主窗口渲染完成。 | 无需处理。 |
| **O103** | Default preset release | 内置默认预设释放到本地 AppData。 | 无需处理。 |
| **O104** | Check update | 触发了自动检查更新。 | 无需处理。 |
| **E101** | App init fail | 软件初始化极其致命失败。 | 检查 Node/Electron 底层依赖是否完整。 |
| **E102** | Preset release fail | 预设释放失败。可能原因：C盘权限不足或磁盘满。 | 建议用户以管理员身份运行，或检查磁盘空间。 |
| **E103** | Auto-updater fail | 检查更新组件报错。 | 检查 GitHub 连通性，或 electron-updater 配置。 |

| 代码 | 定义 (Action) | 说明与可能原因 (Causes) | 排障建议 (Fix) |
| :--- | :--- | :--- | :--- |
| **O201** | Select brand | 选择品牌。参数 `b`=品牌ID。 | 用户正常交互。 |
| **O202** | Select printer | 选择机型。参数 `p`=机型ID。 | 用户正常交互。 |
| **O203** | Select version | 选择版本。参数 `v`=版本ID。 | 用户正常交互。 |

### 模块 3：本地文件读写 (IO - 3xx)
| 代码 | 定义 (Action) | 说明与可能原因 (Causes) | 排障建议 (Fix) |
| :--- | :--- | :--- | :--- |
| **O301** | Read preset | 读取本地预设 JSON。 | 正常行为。 |
| **O302** | Write preset | 修改并保存 JSON 预设。 | 正常行为。 |
| **O304** | Copy model | 从安装目录复制 3MF 到 AppData。 | 正常行为。 |
| **E301** | Preset not found | 尝试读取的 JSON 文件不存在。 | 用户未下载预设或手动删除了文件。引导重新下载。 |
| **E302** | Preset parse err | JSON 格式损坏，无法 Parse。 | 文件被非法篡改。引导用户删除本地预设后重新下载。 |
| **E303** | Preset write err | 无法写入 JSON。可能文件被其他软件占用。 | 提示用户关闭相关文本编辑器或修改只读权限。 |
| **E305** | Model src missing| 找不到软件自带的 3MF 校准模型。 | 软件安装包损坏，建议重新安装软件。 |
| **E306** | Model copy err | 3MF 复制失败。通常是旧模型正被切片软件打开。 | 提示用户先关闭切片软件，或检查 EBUSY 占用的错误。 |
| **E307** | Log write err | 日志写入失败。 | 极低概率，通常是磁盘满了。 |

### 模块 4：网络与云端请求 (NET - 4xx)
| 代码 | 定义 (Action) | 说明与可能原因 (Causes) | 排障建议 (Fix) |
| :--- | :--- | :--- | :--- |
| **O401** | Fetch manifest | 尝试拉取预设清单 `manifest.json`。 | 正常行为。 |
| **O402** | DL preset | 尝试下载具体的 JSON 预设文件。 | 正常行为。 |
| **E401** | Manifest timeout | 清单获取超时。原因：断网、被墙或局域网拦截。 | 让用户检查网络，或开启/关闭代理(TUN)。 |
| **E402** | Manifest parse err| 获取到了数据但不是 JSON。通常是被软路由/广告拦截页劫持。| 让用户切换手机热点测试。 |
| **E403** | DL timeout | 文件下载超时。 | 网络波动，建议重试。 |
| **E404** | DL 404 | 云端仓库里没有这个机型的文件。 | 检查云端 release 仓库是否漏传了对应的 JSON。 |
| **E405** | DL save err | 下载成功但保存到本地失败。 | 磁盘权限问题。 |

### 模块 5：核心数据与引擎 (CORE - 5xx)
| 代码 | 定义 (Action) | 说明与可能原因 (Causes) | 排障建议 (Fix) |
| :--- | :--- | :--- | :--- |
| **O501** | Calc Z offset | Z 轴计算完毕，准备写入。 | 正常行为。 |
| **O503** | CLI process | CLI 模式启动，正在处理 Gcode。 | 正常行为，后台运行中。 |
| **E501** | Offset NaN | 数值计算得到 NaN。致命逻辑错误。 | 检查传入的 offset 字符串是否包含非法字符。 |
| **E502** | Key missing | 在 JSON 中找不到 `toolhead.offset.z`。 | 预设版本太老或结构变更。需适配旧版 JSON 结构。 |
| **E503** | CLI args miss | CLI 模式缺少必要的启动参数。 | 切片软件配置的后处理脚本路径或传参写错了。 |
| **E504** | Gcode IO err | 无法读取或写入 Gcode 文件。 | Gcode 文件被锁定或切片软件未释放句柄。 |
| **E505** | Gcode parse err | Gcode 正则表达式匹配替换失败。 | 切片软件更新了底层 Gcode 语法格式，需更新正则引擎。 |

| :--- | :--- | :--- | :--- |
| **O601** | Open slicer | 呼叫系统默认程序打开 3MF 模型。 | 正常行为。 |

| **E602** | Folder call err | 资源管理器呼叫失败。 | Windows 底层异常，极少见。 |
| **E604** | CLI exec err | 命令行处理进程意外崩溃退出。 | 提取完整的崩溃堆栈分析。 |
