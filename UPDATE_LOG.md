# 🚀 支撑面改善工具 (MKP Support) 更新汇总

## 一、 视觉与交互体验 (UX/UI Polish)

* **优雅的最小动画时长**：为"检查更新"等网络请求增加了强制最小动画时间（如 600ms），解决因本地请求过快导致动画闪烁、用户感知不到刷新动作的问题。
* **下载/应用微交互状态机**：
  * 按钮现在具备完整的状态反馈循环：`点击` -> `下载/应用中(转圈)` -> `成功(变绿打勾)` -> `2秒后自动恢复`。
  * **"已应用"状态解锁**：允许用户重复点击已应用的预设，触发"重新应用"动画并强行刷新底层缓存路径，解决了历史遗留数据导致的路径缺失问题。

* **单体精准呼吸动画**：下载完成后，仅对本地列表中的新增项执行"背景色呼吸 + 微微缩放"淡入效果，不再刷新整个列表，极大减少了视觉跳变。
* **侧边栏布局修正**：彻底修复了侧边栏在折叠状态下图标未居中、文字溢出的 UI Bug，确保了极致的响应式缩放体验。
* **暗黑模式质感调优**：优化了深色模式下的高亮底色和阴影逻辑，解决以往"发白"或边缘裁剪不全的问题，提升了专业感。

## 二、 预设管理架构 (Preset Logic)

* **核心渲染逻辑解耦**：彻底重构了本地列表与在线列表的渲染逻辑，使两者互不干扰。现在点击"应用"、"删除"或展开卡片，下方的在线预设列表将始终保持原位，不会被意外清空。
* **智能状态缓存**：引入在线预设独立记忆体系，仅在切换机型或版本类型时触发清理，提升了在不同版本间切换时的操作连贯性。
* **数据一致性加固**：通过给卡片绑定唯一 `dataset-id`，实现了对 DOM 元素的精准点名修改，彻底杜绝了因闭包记忆或文字匹配导致的重复应用 Bug。

## 三、 校准与工程化功能 (Engineering Tools)

* **后处理脚本路径自动化**：
  * 实现了 EXE 程序路径与 JSON 预设路径的实时自动拼接。
  * 动态生成符合切片软件格式的完整命令字符串，支持一键复制到"后处理脚本"框。

* **只读模型保护机制**：
  * **安全隔离**：核心 3D 模型存储在只读目录，用户点击时自动提取副本至 `AppData/Roaming` 目录。
  * **防占用策略**：若模型被切片软件锁定无法覆盖，系统会自动生成带时间戳的新文件并智能清理旧文件，确保用户永远能打开"干净"的校准模型。
  * **系统调用**：支持首次打开时弹出 Windows"打开方式"选择框，后续则实现无感调用。

* **JSON 全参数映射引擎**：
  * 开发了对象扁平化与逆向还原算法，将深层嵌套的 JSON 配置文件自动映射为 UI 上的参数修改矩阵。
  * 支持在页面上直接修改参数并实时写回本地 JSON，打通了"配置-修改-应用"的全链路。

## 四、 底层稳定性保障 (Core Stability)

* **主进程优化**：彻底去除了 `main.js` 中的重复引入与冲突通道，重理了 IPC 通信顺序，提升了软件启动速度与运行稳定性。
* **引导流程修复**：重构了向导页（Wizard）的生命周期逻辑，修复了过完引导页后配置未保存及 UI 容器位置偏移的致命错误。
